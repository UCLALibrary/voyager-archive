Voyager Archive

The Voyager Archive (VA) contains data from UCLA's Voyager databases.  Data from each Voyager database is preserved in a separate Archive database.  This preserves the Voyager structure, unlike Alma / UCLS which merges data from all 3 Voyager databases into one system.

How to use the Voyager Archive?

The VA user interface is designed for known-item searching.  If something in Alma might not have migrated correctly from Voyager, use the appropriate Voyager identifier to look it up in the VA.

Searching can be done by:

  • MARC record id (the 001 field), for authority, bibliographic, and holdings records
    • Alma records add a suffix to Voyager record ids, like 12345-ucladb .  Search just for the numbers, like 12345 .
  • Item barcode
  • Purchase order number
  • Invoice number
  • Vendor code

If a search finds just one record, the full record will display.  If a search finds multiple records, brief information about each will display.  Click the link to see the full record.

When viewing a record, scroll down to see related data, and click links to navigate between related records.  For example, brief holdings information displays below bibliographic records, and brief item information displays below holdings records.

Click links to move between records, or use the "Search" form at the top of each page to start a new search.

What's in the Voyager Archive?

The VA contains the following data:

  • All MARC records: authority, bibliographic, and holdings (aka mfhd)
  • All item records
  • All item circulation history, including requests and fines/fees
    • This does not include any patron information
  • All locations
  • All purchase orders
  • All invoices
  • All vendors, including account records
    • Vendor addresses were not archived
  • All ledgers and funds
  • All fund transactions
  • Various "look-up" tables, which help translate Voyager's internal data into readable data

What's NOT in the Voyager Archive?

The VA does not contain the following data:

  • Patrons, or any data about / related to them
  • Course reserves
    • Item records have a count of historical reserve charges
  • Circulation calendars and policy matrices
  • Vendor addresses
    • These were difficult to archive, and determined to be of no historical value
  • Permissions / security profiles
  • Headings or other MARC-derived data used for search / discovery
    • Authority and bibliographic MARC records are archived, but not Voyager-derived headings relationships
  • Voyager-specific MARC editing history (the "History tab" in the Voyager cataloging client)
  • Serial patterns and predictive check-in data

For a complete list of all Voyager tables with with notes on purpose and archiving decisions, see Voyager Table Archiving Decisions (UCLA Library staff only).

Questions / suggestions / problems / bugs?

Please use the General staff technology support form to let us know how the Voyager Archive could be improved.  Or, if you need a report on a lot of Voyager data, please provide details and we'll try to help.
